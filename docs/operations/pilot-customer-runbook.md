# Pilot Customer Runbook

## Objetivo
Orientar o teste controlado com 1 cliente piloto antes de escalar.

## Fluxo
1. Selecionar cliente piloto.
2. Criar checkout sandbox ou cobrança real controlada.
3. Validar pagamento.
4. Validar webhook.
5. Validar e-mail de ativação.
6. Cliente define senha.
7. Cliente completa onboarding.
8. Cliente cria primeiro formulário/funil.
9. Suporte acompanha primeiras 48h.
10. Registrar bugs e melhorias.

## Checklist
- [ ] cliente recebeu link correto
- [ ] cliente definiu senha
- [ ] cliente conseguiu login
- [ ] cliente acessou dashboard
- [ ] cliente entendeu fluxo
- [ ] suporte recebeu feedback
- [ ] billing ficou correto
- [ ] admin consegue visualizar tenant
